AGAPAY: Calamity & Disaster Alert System

AGAPAY is a system designed to provide real-time news flashes and emergency alerts to users during calamities and disasters. 
Its main goal is to deliver fast, reliable, and accessible information that helps people stay informed and safe in critical situations.

Objectives:
- Provide real-time alerts for disasters (e.g., typhoons, earthquakes, floods)
- Deliver flash news updates related to ongoing calamities
- Help users stay aware of situations in their area
- Ensure fast and reliable communication during emergencies
  
Features:
- Instant notification system for alerts
- Flash news updates dashboard
- User-friendly interface for easy access
- Emergency alert broadcasting
- Organized information display
  
System Components:
- Frontend (View): User interface for displaying alerts and news
- Backend (Controller): Handles logic, processes alerts, and manages system flow
- Database (Model): Stores user data, alerts, and news updates
  
Technologies Used:
- Java (Core system logic)
- MySQL (Database)
- MVC Architecture (Model-View-Controller design)

How It Works
- The system collects data about disasters or emergencies
- Processes and verifies the information
- Sends alerts and updates to users in real-time
- Displays the information through a simple interface

Use Case
AGAPAY is useful for:
- Communities in disaster-prone areas
- Schools and organizations
- Local government units (LGUs)
- Individuals who need real-time safety updates

Importance
During disasters, timely information can save lives. 
AGAPAY ensures that users receive critical updates quickly, helping them make informed decisions and stay safe.
